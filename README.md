# raptor-comply-docs

Public customer API documentation for [Raptor Comply](https://raptorcomply.com),
published at [docs.raptorcomply.com](https://docs.raptorcomply.com) via Mintlify.

## Structure

| Path | Authored by | Notes |
|---|---|---|
| `introduction.mdx` | Hand-written | Welcome page |
| `authentication.mdx` | Hand-written | API key + Organization header setup |
| `quickstart.mdx` | Hand-written | One curl example |
| `api-reference/` | **Auto-generated** | Do not edit by hand. Generated by `api/scripts/generate-customer-openapi.ts` in [`raptor-comply`](https://github.com/RaptorMaps/raptor-comply) and synced via `auto/api-sync` PRs. |
| `mint.json` | Mixed | Top-level keys hand-edited once; the `"API Reference"` nav group is auto-managed by the sync workflow. |
| `images/` | Hand-managed | Brand assets. |

## Editing

- Hand-written pages: edit on a `feat/` branch, open a PR, merge after CODEOWNERS approval.
- Auto-generated pages under `api-reference/`: do NOT edit directly. Change the upstream
  ts-rest contract or OpenFGA model in `raptor-comply`; the sync workflow regenerates
  these files automatically.

## Branch protection

`main` requires PR + CODEOWNERS approval + stale-approval dismissal on new commits.
Direct pushes and force pushes are disallowed. The auto-sync workflow pushes to a
persistent `auto/api-sync` branch only.

See the [Mintlify API Docs Automation design](https://github.com/RaptorMaps/raptor-comply-os/blob/main/docs/superpowers/specs/2026-05-28-mintlify-api-docs-automation-design.md)
and [rollout plan](https://github.com/RaptorMaps/raptor-comply-os/blob/main/docs/superpowers/plans/2026-05-31-mintlify-api-docs-rollout-plan.md)
for the full architecture.
